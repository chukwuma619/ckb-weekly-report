# CKBuilder weekly progress

Personal dev log for the [CKBuilder](https://nervos.org) programme: one markdown report per completed week, published on GitHub.

## Naming convention

Reports live under `YYYY/mmm/` so each month has its own folder and filenames stay unique week to week.

**Week boundary:** each report covers **Sunday through Saturday** (the next report starts on each **Sunday**).


| Pattern                                | Example                                  |
| -------------------------------------- | ---------------------------------------- |
| `YYYY/mmm/YYYY-MM-DD_to_YYYY-MM-DD.md` | `2026/may/2026-05-03_to_2026-05-09.md`   |
| Next week (Sun–Sat)                    | `2026/may/2026-05-10_to_2026-05-16.md` ( |


Use one file per **reporting week**. Avoid batching multiple weeks into one file

## What to include in each report

Aligned with the CKBuilder handbook: what you did,  evidence (links, commit refs, screenshots if useful), and what you plan next.
